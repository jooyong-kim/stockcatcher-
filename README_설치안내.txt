■ 폴더 구성
+ 스톡캐쳐_VOL17.html ........ 웹앱 본체 (Netlify에 이 파일만 배포해도 됨)
+ pipeline/collect_krx.py ... 전 종목 수집 + 지표 계산 (VOL15 파이프라인: 46종 지표 + 수급 + 종목별 3년 차트, 시가0가드·수급120일)
+ .github/workflows/daily.yml 매일 자동 실행 설정 (평일 19:00 KST)
+ data/ ..................... screener.json이 생성되는 위치
+ data/charts/ .............. 종목별 차트 데이터(120거래일 OHLCV)가 생성되는 위치

■ 설치 순서 (약 15분, 전부 무료)

1️⃣ GitHub에 새 저장소 생성 (Public 권장 — Actions 무료 무제한)

2️⃣ 이 폴더 내용물을 그대로 저장소에 업로드
   (.github 폴더가 숨김 처리되지 않게 주의)

3️⃣ 저장소 Settings → Actions → General →
   Workflow permissions를 "Read and write permissions"로 변경 후 저장

4️⃣ Actions 탭 → "스톡캐쳐 일일 데이터 수집" → Run workflow 버튼으로
   첫 수집을 수동 실행 (약 40~50분 소요 — 3년치 시세 + 수급 수집 포함.
   워크플로우 제한시간은 120분으로 넉넉히 설정되어 있어요)

5️⃣ 완료되면 data/screener.json 생성 확인 후, 파일 화면에서
   Raw 버튼 주소 복사:
   https://raw.githubusercontent.com/계정명/저장소명/main/data/screener.json

6️⃣ 스톡캐쳐 화면 우측 상단 "🧪 데모 시세" 배지 클릭 → 주소 붙여넣기
   → 새로고침되며 "📡 실데이터 연결됨"으로 전환

이후에는 평일 저녁마다 자동 갱신돼요. 수동 갱신이 필요하면
Actions 탭에서 Run workflow를 누르면 돼요.

■ 참고
+ API 키, 회원가입, 증권 계좌 전부 불필요 (pykrx = KRX 공개 데이터)
+ 로컬 PC에서 직접 실행도 가능:
  pip install pykrx pandas numpy
  python pipeline/collect_krx.py
+ 첫 실행이 실패하면 대부분 3번(권한 설정) 누락이에요.
+ VOL13부터 종목별 차트 파일(약 2,000여 개)이 함께 커밋돼요. 매일 갱신되며
  저장소 용량이 서서히 늘어나니, 몇 달에 한 번 히스토리 정리를 권해요.
+ 차트는 종목을 눌렀을 때 해당 파일만 그때그때 불러와서 앱은 계속 가벼워요.

■ 장기 아카이브 수집기 (pipeline/archive_krx.py) — 선택 기능

목적: 백테스트용으로 종목별 전체 히스토리(상장 초기까지)를 로컬에 수집

실행 (로컬 Mac, pipeline 폴더에서):
  export KRX_ID='아이디'
  export KRX_PW='비밀번호'
  python3 archive_krx.py

+ 현재 → 과거 방향으로 3년 청크씩 수집, 실행할 때마다 이어서 진행돼요
+ 상장 초기까지 다 모이면 종목별로 "🏁 상장 초기까지 수집 완료"라고 알려줘요
+ 저장: data_archive/ 폴더 (종목별 CSV + 진행상태 json)
+ 주의: 전체 용량이 수백 MB까지 커질 수 있어서 GitHub에는 올리지 마세요
  (.gitignore에 이미 제외 처리되어 있어요)
+ 옵션: --minutes 120 (2시간 수집) / --limit 100 (100종목만)

+ 차트 상단에서 일봉/주봉/월봉 전환 (일봉 데이터를 화면에서 즉시 리샘플링)
